# Reference Rules

- *ALWAYS* write austere, neutral descriptions of the machinery! Reference is consulted, not read.
- *ALWAYS* be authoritative! There should be no doubt or ambiguity in reference material.
- *ALWAYS* adopt consistent standard patterns! Consistency is what makes reference material useful.
- *ALWAYS* respect the structure of the machinery! The documentation structure should mirror the product structure.
- *ALWAYS* provide examples of usage to illustrate without becoming instructional or explanatory.
- *ALWAYS* state facts about the machinery and its behaviour clearly and precisely.
- *ALWAYS* list commands, options, flags, limitations, and error messages in an orderly way.
- *NEVER* explain why! Explanation belongs in explanation documentation. Link to it instead.
- *NEVER* give instructions! How-to content belongs in how-to guides. Link to them instead.
- *NEVER* introduce style, personality, or opinion! Reference is not the place for creativity.
