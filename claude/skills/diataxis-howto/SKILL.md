# How-to Guide Rules

- *ALWAYS* write goal-oriented content addressed to a real-world problem, not to a tool! The reader knows what they want and needs directions to get there.
- *ALWAYS* choose titles that say exactly what the guide shows: "How to integrate..." not "Integrating..."
- *ALWAYS* address real-world complexity! A guide useful for only one exact case is rarely valuable.
- *ALWAYS* seek flow! Ground sequences in the pattern of the user's activities so progress feels smooth.
- *ALWAYS* start and end in meaningful places. Practical usability is more helpful than completeness.
- *ALWAYS* assume the reader is already competent! They know what they want to do.
- *NEVER* teach or explain! If explanation is needed, link to it.
- *NEVER* provide reference material! Direct the reader to the reference guide for full lists of options.
- *NEVER* write from the perspective of the machinery! Write from the perspective of the user's need.
- *NEVER* include unnecessary setup steps! Omit what the competent user should already know.
